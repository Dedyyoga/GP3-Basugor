Aplikasi secara garis besar;
Terdapat sebuah API yang menyimpan informasi motor yang akan dikoleksi, dimana User dapat memilih motor untuk dikoleksi

Tentang Table:
1. User / Owner dapat memilih banyak motor yang akan dikoleksi
2. Banyak Motor dapat dipilih oleh banyak user
3. Motor dapat memiliki 1 kategori
4. 1 kategori bisa dimiliki banyak motor
5. Motor dapat memiliki 1 kreator (pabrikan)
6. 1 pabrikan dapat memiliki banyak motor


PERTANYAAN:
1. Apakah make sense jika user melakukan method Update Koleksi? Boleh
2. Apakah kategori boleh di create oleh admin? Boleh
3. Apakah creator boleh di create oleh admin? Boleh
4. Method yang ada pada Kategori apakah hanya read dan create? Update (Admin) / !ON DELETE CASCADE
5. Apakah method update dan delete ada pada creator? Update Bisa / !ON DELETE CASCADE


BREAKDOWN PENGERJAAN:
1. CRD (User)
2. CRUD (Motor) - CUD (Admin)
3. CRUD (Kategori) - Admin ? R - User 
4. CRUD (Creator) - Admin ? R - User


1. /users (GetAll(Admin), GetDetail(Own), Patch(Own))

2. /motors (GetAll(User, Admin), GetDetail(User, Admin), Post(Admin), Delete(Admin), Patch(Admin))

3. /creators (GetAll(User, Admin), GetDetail(User, Admin), Post(Admin), Delete(Admin), Patch(Admin))

4. /categories (GetAll(User, Admin), GetDetail(User, Admin), Post(Admin), Delete(Admin), Patch(Admin))

3. /volumes (GetAll(User, Admin), GetDetail(User, Admin), Post(Admin), Delete(Admin), Patch(Admin))

4. /users/signup

5. /users/login
